PERSONAL SYSTEM · ARCHITECTURE NOTES · JULY 2026

One file, two agents, zero shared code.

Getting interview-ready in HLD and LLD turned into a small systems problem: one agent needed to watch what I actually do all day, and another needed to judge whether it's working. Rather than merge them, I split them clean down the middle and gave them exactly one thing in common.

03 — Ground rules

Why it's built this way.

Boundary

Data-only autonomy

The local agent can append to the diary, read it, and poll Todoist. It cannot edit its own code, run shell commands, or touch any other file — enforced by which functions exist, not by asking the model nicely.

Reliability

Never destructive

Auto-refine only replaces the diary's raw entries if the local model returns a real, sane result. Unreachable Ollama, garbage output, anything under 8 words — it silently keeps your raw notes instead.

Reliability

Deterministic core, LLM garnish

The tracker's coding-activity and study-progress sections are plain code, never LLM-dependent. Only the closing feedback paragraph touches an LLM — and it falls back to plain text if that output looks broken.

Separation

One shared file, not shared code

No imports between the two projects. The tracker doesn't know the local agent exists; the local agent doesn't judge progress. The diary file is the entire interface.
